3. Ventajas

Contenido

Separación de componentes conceptuales y lógicos

Los usuarios cuando realizan una consulta en una Base de Datos Relacional, navegan a través de ella por medio de un esquema con tablas de registros, logran obtener los datos requeridos mediante operaciones asociadas como juntura y proyección que son las más comunes.

Sobrecarga semántica

Una relación es usada normalmente en el Modelo Relacional, muchos tipos diferentes de información, la existencia de entidades implícitas, propiedades, estructura y asociaciones entre ellos. El modelo semántico de datos provee una separación de esta información, mediante la abstracción de datos.

Abstracción de datos

Esto permite proveer de datos derivados, que están relacionados más a la noción que tiene el usuario acerca de las Bases de Datos. Estos datos no están incorporados al esquema, es decir es un nuevo tipo de dato (puede tomarse como una entidad nueva en una relación ISA o relación por herencia, por ejemplo).
